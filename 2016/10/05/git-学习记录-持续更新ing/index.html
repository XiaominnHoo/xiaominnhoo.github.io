平时工作中使用 git, 也就只会用常用的那几个命令，对 git 了解甚少，很多概念都了解不清晰，所以，有空就学习下 git，以此对某些概念更清晰，并以此做记录。

origin/HEAD

Q: 在执行git branch -r 查看远程分支命令时，有个 origin/HEAD -> origin/master

A: 表示clone repo 时会默认指定master 为checkout 的分支，可以通过命令 git remote set-head origin **设置改变默认 checkout 的分支

HEAD

当前分支版本顶端的别名, 即在 当前分支的git 提交历史记录中，指向当前所在的位置，一般是指向最近的一次 commit， 但也可以是历史提交记录中的 某次commit

撤销操作

git commit –amend

使用场景是：某次提交，提交信息写错了，或是忘记提交了几个文件；

  1. 如果提交后，没有任何改动，运行此命令，编辑提交说明，保存退出后，新的提交说明就会覆盖刚才的提交说明。
  2. 如果提交后，添加忘记提交的文件到暂存区，再运行此命令，提交的内容就变成第一次 commit 的内容和 ammend 提交的内容了。

git reset HEAD file_name

取消文件暂存

git checkout – file_name

取消对文件的修改

改写提交（重置操作）

reset命令做的事情就是重置HEAD(当前分支的版本顶端）到另外一个commit

git reset –hard head^

重置最新的1次 commit: 也就是丢弃上次提交的修改，就像没有提交过一样。

使用 git reset –hard命令要小心，保证执行此命令前的工作目录是干净的，否则可能会把工作目录中修改的内容也丢弃了。

补充：

  1. git reset –hard head：会把当前工作目录中所有未提交的内容清空，让工作目录回到上次提交时的状态。
  2. head^ 中的^的表示head 之前的一个 commit, 等价于 head~1
  3. head~n: 表示 head 之前的n 个 commit。

git reset –soft head^

这条命令和–hard 的区别是：不会丢弃上次提交的修改，上次提交的修改会在缓存（stage）区域。
